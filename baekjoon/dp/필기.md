# 핵심은 3개

### 1. 상태 정의

dp[i], dp[i][j]가 무슨 뜻인지 정하는 것.
예를 들면

- dp[i] = i번째까지 왔을 때 최대값
- dp[i] = i를 만드는 최소 연산 횟수
- dp[i][j] = i개와 j개가 있을 때 경우의 수

즉, dp 배열 한 칸이 의미하는 바를 정하는 거야.

### 2. 점화식

즉, 점화식은 현재 문제 = 더 작은 문제들의 조합

### 3. 초기값

점화식은 시작점이 있어야 돌아가.

예를 들어 피보나치에서

- dp[1] = 1

- dp[2] = 1

이게 없으면 dp[3]도 계산 못 해.

즉 초기값은
가장 작은 문제의 정답이야.

---

**DP 문제를 보면 이 4개를 꼭 적어봐.**

- dp[i] 또는 dp[i][j]의 뜻
- 초기값
- 점화식
- 왜 그런지 말로 설명

**문제마다 아래 형식으로 적어.**

> 1. 상태 정의
> dp[i] = ...

> 2. 점화식
> dp[i] = ...

> 3. 초기값
> dp[0] = ..., dp[1] = ...

> 4. 계산 방향
> 작은 값 → 큰 값

### DP 는 주로 유형이 있음

- 경우의 수 세기
- 최대/최소값 구하기
- 선택/비선택
- 앞에서부터 누적
- 배낭 문제
- 문자열 DP
- 증가 부분 수열

### 입문 순서 추천

- 피보나치
- 1로 만들기
- 계단 오르기
- 1010 다리 놓기
- 퇴사
- RGB 거리
- 포도주 시식
- LIS 기초
